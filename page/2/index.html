Ubuntu 用户主目录文件夹名称改为英文1234567891011121314# 语言环境改为英文export LANG=en_US# 弹出对话框，问是否改成英文，点是，并且选“不再提示”xdg-user-dirs-gtk-update# 语言环境改回中文export LANG=zh_CN# 这样的话需要重新登录，推荐这样弄sudo apt-get install tmuxtmuxexport LANG=en_USxdg-user-dirs-gtk-updateexit这样就不用重启了 让 xdg-user-dirs-gtk-update 可以重新执行 自从上次使用xdg-user-dirs-gtk-update点击了不在问我这个问题后，再次执行这个命令就会没有反映，现在总算找到了解决方法。 执行echo zh_CN > ~/.config/user-dirs.locale命令，然后就可以在次执行 xdg-user-dirs-gtk-update 了
