Git 权威指南

  • 版本控制是管理数据变更的艺术
  • 版本控制能决定项目的成败，甚至是公司的生死
  • 唯有了解真相（Git 原理），才有可能自由（掌握 Git）
  • 1991 年-2002 年这十余年间，Linus 宁可手工修补文件也不愿意使用 CVS。2002 年-2005 年，Linus 选择了一个商业版本控制系统 BitKeeper 作为 Linux 内核的代码管理工具。
  • 2005 年发生的一件事最终导致 Git 诞生。在 2005 年 4 月，Andrew Tridgell（大名鼎鼎的 Samba 作者）试图对 BitKeeper 进行反向工程，以开发一个能与 BitKeeper 交互的开源工具。这激怒了 BitKeeper 的公司 BitMover，并收回对 Lunux 社区免费的 BitKeeper 授权。迫不得已，Linus 选择自己开发一个分布式控制工具，Git 由此诞生。
  • 安装 gitsudo apt-get install git git-doc git-svn git-email git-gui gitk

简易 Git 命令行入门教程

  • Git 全局设置:
    git config –global user.name “侠义书生”
    git config –global user.email “xiayishusheng@gmail.com“

  • 创建 git 仓库:
    mkdir OkHttpProj
    cd OkHttpProj
    git init
    touch README.md
    git add README.md
    git commit -m “first commit”
    git remote add origin https://git.oschina.net/xiayishusheng/OkHttpProj.git
    git push -u origin master

  • 已有项目
    cd existing_git_repo
    git remote add origin https://git.oschina.net/xiayishusheng/OkHttpProj.git
    git push -u origin master

  • git 无法 pull 仓库 refusing to merge unrelated histories
    假如我们的源是 origin，分支是 master，这样写 git pull origin master –allow-unrelated-histories
